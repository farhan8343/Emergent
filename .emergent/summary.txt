<analysis>**original_problem_statement:**
Build a SaaS web application called Markuply for visual markup and review. Users (Owners, Team Members, and Guests) should be able to leave pin-based comments on live website URLs, PDFs, and images. The application must include a subscription system with different plans (Starter, Pro, Business) that limit team members and storage. An admin dashboard is required for the account owner to manage users, subscriptions, and view usage statistics. The commenting system should support threaded discussions, file attachments, and status tracking (open/resolved). The original tech stack requested was React, Node.js (Express), PostgreSQL, and Stripe.

**User's preferred language**: English

**what currently exists?**
The application is being built with an agreed-upon stack of **FastAPI, React, and MongoDB**. The initial MVP has skipped payment integration (Stripe) and uses local server storage for files. A JWT-based authentication system for registered users and a name/email flow for guests have been implemented.

The core functionality has gone through several major iterations. The initial -based approach for displaying live websites was abandoned due to fundamental CORS and security issues. The current architecture uses a server-side screenshot of the target URL, which is then displayed on a canvas for users to add pins and comments.
